## Problem Statement

Given a set of student attributes such as gender, race/ethnicity, parental level of education, lunch type, and test preparation course completion — along with their reading and writing scores — predict their **math score**.

This is a **regression problem** evaluated using the R² (coefficient of determination) metric.

## Pipeline Overview

The training pipeline runs through three sequential stages:

### 1. Data Ingestion (`data_ingestion.py`)
- Reads `stud.csv` from `notebook/data/`
- Saves a full copy to `artifacts/data.csv`
- Splits data into 80% train / 20% test using `train_test_split`
- Saves splits to `artifacts/train.csv` and `artifacts/test.csv`

### 2. Data Transformation (`data_transformation.py`)
- **Numerical features** (`reading_score`, `writing_score`):
  - Median imputation → Standard scaling
- **Categorical features** (`gender`, `race_ethnicity`, `parental_level_of_education`, `lunch`, `test_preparation_course`):
  - Most-frequent imputation → One-hot encoding → Standard scaling
- Builds a `ColumnTransformer` preprocessing pipeline
- Saves the fitted preprocessor to `artifacts/preprocessor.pkl`

### 3. Model Training (`model_trainer.py`)
- Trains 9 regression models with GridSearchCV hyperparameter tuning (3-fold CV)
- Selects the best model based on R² score on the test set
- Raises an exception if no model achieves R² ≥ 0.6
- Saves the best model to `artifacts/model.pkl`

## Hyperparameter Tuning

`GridSearchCV` (3-fold cross-validation) is applied to all models that have tunable parameters. The `evaluate_models` function in `src/utils.py` handles this automatically.

| Model | Parameters Tuned |
|---|---|
| Random Forest | `n_estimators`, `max_depth`, `min_samples_split` |
| Gradient Boosting | `n_estimators`, `learning_rate`, `max_depth` |
| AdaBoost | `n_estimators`, `learning_rate` |
| CatBoost | `iterations`, `learning_rate`, `depth` |
| XGBoost | `n_estimators`, `learning_rate`, `max_depth` |
| KNN | `n_neighbors`, `weights` |
| Decision Tree | `max_depth`, `min_samples_split`, `criterion` |
| Logistic Regression | `C`, `max_iter`, `solver` |
| Linear Regression | *(no hyperparameters tuned)* |

## How to Run

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd "ML Project"
```

### 2. Create and activate a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # macOS/Linux
venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

> The `setup.py` file registers `src` as an installable package (`mlproject`). If you want an editable install, uncomment `-e .` in `requirements.txt` and run `pip install -e .`.

### 4. Run the training pipeline

```bash
python src/components/data_ingestion.py
```

This triggers the full pipeline: ingestion → transformation → model training. Artifacts are saved in the `artifacts/` folder.

## Logging & Exception Handling

- **Logging:** Every major step is logged via a centralized logger defined in `src/logger.py`. Log files are timestamped and stored in the `logs/` directory.
- **Exception Handling:** A custom `CustomException` class in `src/exception.py` wraps all exceptions with detailed traceback information including the filename and line number where the error occurred.
